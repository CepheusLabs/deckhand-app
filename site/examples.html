Recipes.

Common end-to-end runs. Each one shows the GUI path and the shell-only version. Use whichever you like.

Phrozen Arco: stock keep with Kalico + Mainsail

The most common path. The vendor's OS sticks around, the vendor's services don't, you end up on Kalico and Mainsail with KIAUH installed for next time.

In Deckhand
  1. S20, pick the discovered Arco (or paste IP)
  2. S40, Stock keep
  3. S145, snapshot all paths
  4. S100, Kalico
  5. S105, Mainsail
  6. S107, install KIAUH
  7. S110, KlipperScreen
  8. S120, disable makerbase-net-mods, phrozen-screen-svc, makerbase-cloud
  9. S140, tick all leftover files (defaults are sane)
  10. S150, change SSH password, fix time-sync
  11. S900, Run · ~12 min
Headless CLI
cat > arco-plan.yaml <<EOF
path: stock_keep
firmware: kalico
webui: mainsail
kiauh: true
screen: klipperscreen
services_disable:
  - makerbase-net-mods
  - phrozen-screen-svc
  - makerbase-cloud
files_remove: all_defaults
hardening:
  ssh_password: $(read -s p; echo $p)
  fix_timesync: true
EOF

deckhand install \
  --profile phrozen-arco \
  --host arco.local --user mks \
  --plan arco-plan.yaml --yes

Sovol SV08 Max: fresh flash

Vendor OS is wedged, or you just want a clean base. Pull the eMMC card out, flash it on your laptop, slot it back in, finish over SSH.

  1. Power off the printer, eject the eMMC card.
  2. Plug card into your laptop.
  3. Open Deckhand → Manage view → "Fresh-flash from card".
  4. Pick Armbian image (Deckhand fetches from dl.armbian.com with verified signatures).
  5. Confirm the disk path on S200, Deckhand greys out non-removable disks.
  6. S220 writes ~1.4 GB at native speed; ETA shown live.
  7. Eject card, return to printer, boot, wait 90 seconds for first-boot expansion.
  8. Back in Deckhand → S20 discovers the new printer (now on DHCP) → run the rest of the wizard as Stock keep.

Restore from a snapshot

You ran an install yesterday and the printer's acting up. You want yesterday's printer.cfg back without redoing the whole install.

In Deckhand
  1. Manage view → Backups
  2. Pick the snapshot row (sorted newest first)
  3. Restore → Side-by-side
  4. Open the diff viewer, copy the lines you want
Shell
cd ~/.deckhand/snapshots/
ls -lh phrozen-arco-*.tar.gz

deckhand restore \
  --host arco.local \
  --archive phrozen-arco-2026-04-25.tar.gz \
  --strategy side-by-side
# Restored to ~/printer_data.stock-2026-04-25/ on the printer.

Capturing a debug bundle for a bug report

Before opening an issue, grab one of these. Bundles are auto-redacted (SSH keys, IPs, hostnames, MACs, anything matching the secret regex) and you preview the redacted version before saving the zip.

deckhand debug-bundle --session last --out ./bundle.zip --review

# OR via the GUI: Manage → Debug bundle → Build → Review → Save.
# Attach the .zip to your GitHub issue.

Re-running just one step

Mainsail got mangled. You want to reinstall just it, not redo the whole wizard.

  1. Manage view → Sessions → pick last session
  2. Right-click the mainsail_install step → "Re-run only this step"
  3. Confirm, Deckhand re-evaluates the precheck (already-installed → will be replaced) and runs.

CLI equivalent:

deckhand step run --session 7c44f1 --step mainsail_install
