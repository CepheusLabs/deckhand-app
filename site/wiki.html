The Deckhand wiki.

Deckhand swaps the vendor firmware on your Klipper-based printer for real Klipper, or for Kalico if you want the spicy fork. You can drive it from the GUI or do every step in a shell. This wiki covers both, plus all the bits that aren't either, like profile authoring and the JSON-RPC interface the GUI talks to.

First time here?

Go to Getting started. If you already SSH into your printers and don't want a GUI in your way, the Manual playbook is the same install rendered as shell commands.

What it is, what it isn't

Deckhand is a setup tool. It does the messy first-time install, the vendor cleanup, the snapshots, and the web UI wiring. After that, it gets out of your way. It doesn't try to be your update manager, your slicer, or your printer farm controller, those things already exist and they're better at their jobs. KIAUH handles updates, Mainsail or Fluidd handle prints, Printdeck handles fleets. Deckhand just gets you to a clean starting point and offers to install KIAUH on the way out.

It does

  • Find your printer on the LAN
  • Flash Armbian to the eMMC if you ask for it
  • Install Klipper or Kalico, plus Moonraker
  • Install Mainsail, Fluidd, or both
  • Snapshot and restore your printer.cfg
  • Evict the vendor services you don't want
  • Optional hardening (SSH password, time-sync mess)
  • Make a redacted debug bundle when something goes wrong

It doesn't

  • Track ongoing Klipper updates, KIAUH does that
  • Run prints, that's Mainsail or Fluidd's job
  • Phone home. No telemetry, no accounts, no anything
  • Touch the values inside your printer.cfg
  • Reflash board firmware or bootloaders
  • Work over a USB cable. It's LAN only.

Two ways through it

You pick one of these on screen S40. The wizard rearranges the rest of the flow around your choice.

Path A: Stock keep

Keeps the vendor's OS, swaps the stack on top of it. Snapshots ~/printer_data/, deletes the services and files you tick, drops Klipper or Kalico in alongside the vendor binary (or replacing it). About 10 minutes. Lower stakes, since the snapshot puts you back if you change your mind. Pick this when the printer works fine and you just want a clean stack.

Path B: Fresh flash

Wipes the eMMC and writes a clean Armbian image. About 25 minutes, and yes, if you pull power mid-flash you brick it. The upside is you get a known-good base with zero vendor weirdness. Pick this when the OS is already in a bad state, or when you want the cleanest possible starting point and don't mind the wait.

Words you'll see a lot

The Klipper world has a lot of names for things. Here's what they mean in Deckhand's docs.

TermWhat it is
KlipperThe original firmware project. Slow-moving, well-documented, what guides assume.
KalicoKlipper fork with better input shaping and broader board support. Worth trying if you have a printer that moves fast.
MoonrakerThe API layer between Klipper and any web UI. You don't interact with it directly, but everything else does.
Mainsail / FluiddThe two competing web UIs. Both talk to Moonraker. Install one, install both, doesn't matter, they coexist on different ports.
KIAUHKlipper Installation And Update Helper. An interactive shell menu for the ongoing maintenance Deckhand doesn't do.
ProfileYAML in deckhand-profiles describing what to evict, what to recommend, and what extra steps a printer needs.
SidecarSmall Go binary that does the privileged stuff (writing disks, files). The Flutter UI talks to it over JSON-RPC.
DoctorThe preflight check on screen S10. Pokes your laptop for missing tools and network access before you commit to anything.
SnapshotA tar.gz of your printer's ~/printer_data/, taken before any destructive step. Lives on your laptop, not the printer.
Run-stateA small file on the printer (~/.deckhand/run-state.json) that lets a half-finished run pick back up if you lose power or close the lid.

How the rest of this is laid out

  • Getting started: download, install, run the wizard once.
  • Usage guide: every wizard screen, the Manage tab, debug bundles.
  • Manual playbook ⭐: the same install as shell commands, for when you want to know what's actually happening.
  • CLI & IPC: the deckhand binary and the JSON-RPC catalog the sidecar speaks.
  • Recipes: real workflows. Arco from stock, SV08 from scratch, restoring a backup.
  • FAQ: questions I get asked on Discord enough times to write down.
