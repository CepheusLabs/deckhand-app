FAQ

Things people ask.

If your question isn't here, try discussions or open an issue. Bug reports go a lot faster with a debug bundle attached.

General

Why not just use KIAUH?

KIAUH is great. Deckhand installs it for you on the way out. The thing is, KIAUH assumes you've already got a Klipper-friendly OS to work on, which is exactly the part the vendor took away from you. Deckhand handles that part: kicking out vendor services, snapshotting your config, optionally re-flashing the eMMC. Then KIAUH takes it from there.

Kalico or Klipper?

If you don't have a strong opinion: Kalico. Better input shaping, more boards work out of the box, and the few extra features are opt-in. Pick mainline Klipper if you want the option every guide on the internet assumes, or if you're nervous about running a fork. Either way you can swap later, both store config in ~/printer_data/ and the snapshot survives the change.

Will this brick my printer?

Path A (stock keep) won't. The snapshot puts you back if anything goes wrong. Path B (fresh flash) can, if power cuts mid-write the eMMC ends up in a bad state. Plug into a UPS, or pop the card and flash it on your laptop instead (see SV08 fresh flash). I haven't bricked one yet, but I also haven't tested every printer.

Connection & discovery

It can't find my printer.

Almost always mDNS, your router or firewall is dropping it. Doctor's mdns_resolvable check confirms. Easiest fix: hit "Add manually" on S20 and paste the IP. You can grab it from your router's DHCP leases page or with arp -a after pinging.

"Connection refused" when I enter SSH credentials.

Two usual suspects:

  • Vendor SSH is on a weird port. Some Sovol firmware uses 2222. Try user@host:2222.
  • Vendor SSH is off by default. Turn it on from the touchscreen settings or the vendor app first.

Profile match is "ambiguous."

Two profiles think they're yours. Pick the more specific one from the shortlist. If neither is right, the generic profile works for path B (fresh flash). It won't help much with path A because it doesn't know your specific vendor service mess.

Install issues

"Will build Python 3.11 from source (~20 min)."

Your stock OS ships an older Python than the firmware project needs (3.10 or 3.11 minimum, depending). Deckhand builds a new one in a side directory. The system Python doesn't get touched. The 20 minutes is unattended, go do something else.

A step failed mid-run.

The wizard pauses and offers Retry / Skip / Cancel. Steps are idempotent (how), so retrying is safe. Skip moves past it. Cancel leaves everything done so far in place so you can come back to it.

My laptop went to sleep halfway through.

That's fine. The run-state file on the printer survives. Open Deckhand again and click the "Resume session" banner.

Privacy & security

Does Deckhand send anything to your servers?

I don't have any servers. The app talks to your printer, GitHub (for the firmware), and Armbian's CDN (for OS images on fresh flash). That's the whole list, and you can read it in config/network.yaml.

Are debug bundles safe to share?

The redaction pipeline strips IPs, hostnames, MACs, SSH keys, and whatever else matches the configurable secret regex. You see the redacted preview before the zip gets written. If something looks like it shouldn't go out, it doesn't. The bundle never leaves your laptop on its own. Details in DEBUG-BUNDLES.md.

How do you verify profile updates?

Profiles are signed. The bundled keyring verifies the signature before installing. If a signature doesn't check out, the update is rejected with a banner, never silently. Key rotation is in PROFILE-TRUST.md.

Advanced

Can I script it?

Yes. The CLI wraps the same operations the GUI uses, and the sidecar's JSON-RPC API is open. The HITL test rig drives Deckhand entirely headlessly using exactly that, so the path is well-trodden.

Can I run it offline?

Mostly. Path A works with cached profiles, but the Klipper/Kalico clones still need GitHub. For a fully offline install, mirror the firmware repos locally and point Deckhand at them with --mirror. See Offline rebuild.

Can I add my printer?

Please do. Profiles are YAML in deckhand-profiles. The authoring guide covers the schema and the step DSL. PRs welcome, and so are issues if the schema gets in your way.
