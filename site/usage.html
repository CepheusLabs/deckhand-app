Reference

Using the GUI.

Every screen, what it asks, what it actually does. The wizard handles install. The Manage view is where the rest lives, backups, debug bundles, profile updates.

Anatomy of the wizard window

  • Left rail, the step list. Greyed steps are skipped by your earlier choices.
  • Header strip, printer name, profile version, time estimate.
  • Main panel, the question this screen is asking.
  • Footer, Back / Continue / "Why this question?" link.
  • Right rail (during run), live step status with expandable command logs.

Hit ? anywhere to see the keyboard map. ⌘, opens Settings.

Screen-by-screen reference

S10: Welcome & Doctor preflight

Runs the doctor checks and shows a status strip. "View report" opens the full list. You can Continue anyway past warns; hard fails block the flash screens but not the rest of the app.

S20: Discover

mDNS list of printers on your LAN. Empty list usually means a firewall ate the discovery packet, hit "Add manually" and paste an IP. SSH creds get tested inline before you commit.

S30: Profile match

Deckhand SSHes in, reads /etc/os-release and a few sentinel files, and matches against the profile registry. If two profiles claim your printer you pick from a shortlist. If no match, the wizard offers to use a generic profile (limited features) or abort.

S40: Choose path

Two big cards: Stock keep (transform in place) or Fresh flash (wipe eMMC). The card you can't pick is greyed with a reason, fresh flash needs an eMMC the printer can boot without.

S100: Firmware

Cards from firmware.choices[]. Kalico is the default recommendation on most profiles. If your stock Python is too old, you'll see a "will build Python 3.11 from source (~20 min)" banner so the long step isn't a surprise.

S105 / S107 / S110: Web UI, KIAUH, screen daemon

One card-pick screen each. The KIAUH blurb is worth reading if you've never used Klipper before. KIAUH is what you'll reach for next time you want to add another Klipper instance, swap branches, or reinstall Moonraker.

S120: Per-service questions

One screen per vendor service that has a wizard: block in the profile. The default answer is what the profile recommends; the side panel explains what the service actually does. One question at a time on purpose, each one gets its own page.

S140: Cleanup files

One scrollable list of checkboxes. "Diff preview" expands to show exactly what'll be deleted, with globs resolved to real paths. Defaults come from the profile.

S145: Snapshot (stock keep only)

Live size estimate as you tick paths. Two restore strategies: side-by-side (default, safer) or auto-merge non-conflicting. The tarball stays on your laptop, not the printer.

S150: Hardening

Optional. All boxes start unchecked. Worth doing: change the well-known SSH password, sort out the three-way time-sync mess vendors leave behind, kill UDP services that aren't doing anything for you.

S900: Run

This is the long one. Streams every step's output. Right rail shows status pills as steps go through them. Click any step to see the full command log. If a step fails, the run pauses and you get retry / skip / cancel.

S910: Done

Summary of what changed: services removed, packages installed, files deleted, anything that needs your follow-up. One button to open the new web UI.

Manage view

Lives in the title bar dropdown once you've finished a run. Everything that isn't a fresh install lives here.

Backups

Every snapshot you've taken, listed. Restore in place, export as tarball, or delete. SHA-256 per snapshot.

Debug bundle

Collects logs and run-state, then makes you read the redacted version before it writes the zip. Nothing leaves your laptop on its own.

Profile updates

Pulls the latest profiles from deckhand-profiles. Signature checked. Key rotation handled.

Reinstall step

Re-run a single step from an earlier run. Handy when Mainsail gets corrupted and you just want to put it back.

Settings

  • General, theme, language, default printer.
  • Network, GitHub PAT (raises rate limit), proxy, allow-list audit.
  • Advanced, re-run preflight, sidecar log level, "show profile YAML" toggle.
  • About, versions, license, attributions.
