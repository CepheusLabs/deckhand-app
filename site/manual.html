Audit · script · DIY

Doing it by hand.

Every wizard screen is a few shell commands wearing a UI. Here they are, in order. Useful if you want to audit what Deckhand actually runs, script your own variant, or just skip the GUI. Tested on Phrozen Arco running stock vendor OS; commands are similar on Sovol but paths differ, check your profile YAML.

No safety net here

The GUI takes a snapshot before anything destructive and picks itself up if it crashes. Doing this by hand, that's on you. Start with the snapshot in §2 even if you skip everything else.

1. Preflight: what Doctor checks

Mirror the doctor.run checks before you start:

# Check elevation tool
which pkexec      # Linux
which osascript   # macOS, built in
where powershell  # Windows

# Check disk enumeration works
lsblk             # Linux
diskutil list     # macOS
Get-Disk          # Windows PowerShell

# Check mDNS works
avahi-browse -art          # Linux  (or use `dns-sd -B _ssh._tcp` on mac)
ping printer.local

# Check GitHub reachability + rate limit
curl -s https://api.github.com/rate_limit | jq .rate

# Check clock skew (matters for TLS)
curl -sI https://api.github.com | grep -i ^date

2. Find your printer

The GUI tries mDNS first, falls back to ARP. By hand:

# Browse for SSH services on the LAN
avahi-browse -rt _ssh._tcp     # Linux
dns-sd -B _ssh._tcp .          # macOS

# Or scan the local /24
nmap -sn 192.168.1.0/24

# Once you have an IP, confirm SSH works
ssh mks@192.168.1.42           # Phrozen Arco default
# password: makerbase

3. Snapshot before you touch anything

Same thing S145 does. Streams a tar.gz back to your laptop. Nothing gets left on the printer.

Deckhand
  1. S145 → tick paths to snapshot
  2. "Snapshot and continue"
  3. tar lives in ~/.deckhand/snapshots/
Shell
STAMP=$(date +%Y%m%d-%H%M%S)
ssh mks@printer.local \
  'tar -czf - ~/printer_data ~/klippy_extras 2>/dev/null' \
  > deckhand-snapshot-${STAMP}.tar.gz
sha256sum deckhand-snapshot-${STAMP}.tar.gz

4. Install firmware (Klipper or Kalico)

What S100 does, behind the scenes: a git clone and the project's install script. Pick one.

Kalico (the fork)

ssh mks@printer.local
sudo systemctl stop klipper 2>/dev/null

# Back up any existing klipper dir
[ -d ~/klipper ] && mv ~/klipper ~/klipper.vendor-$(date +%s)

git clone https://github.com/KalicoCrew/kalico.git ~/klipper
cd ~/klipper
./scripts/install-debian.sh
# Reads python_min from setup; will offer to build Python 3.11 from
# source if your distro is too old (~20 min on a CB1).

Klipper (mainline)

git clone https://github.com/Klipper3d/klipper.git ~/klipper
cd ~/klipper
./scripts/install-debian.sh

Then Moonraker (always needed)

git clone https://github.com/Arksine/moonraker.git ~/moonraker
cd ~/moonraker
./scripts/install-moonraker.sh

5. Install a web UI

This is S105. Mainsail and Fluidd are both static bundles, served by nginx.

Deckhand
  1. S105 → pick Mainsail / Fluidd / both
  2. Wizard fetches release tarball
  3. nginx config installed
Shell, Mainsail
sudo apt install -y nginx
mkdir -p ~/mainsail && cd ~/mainsail
wget -q https://github.com/mainsail-crew/mainsail/releases/latest/download/mainsail.zip
unzip -q mainsail.zip && rm mainsail.zip

# Drop the nginx config from Moonraker's docs
sudo cp ~/moonraker/docs/nginx_configs/mainsail \
        /etc/nginx/sites-available/mainsail
sudo ln -sf /etc/nginx/sites-available/mainsail \
            /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

For Fluidd, swap mainsail.zip for fluidd.zip from fluidd-core/fluidd, same shape. Picking both just means installing both bundles on different ports (default 80 + 8080).

6. Drop in KIAUH

S107. One git clone. KIAUH is the menu you'll come back to later for updates.

git clone https://github.com/dw-0/kiauh.git ~/kiauh
~/kiauh/kiauh.sh    # interactive menu

7. Touchscreen daemon (optional)

S110 picks one based on your hardware. For an Arco with KlipperScreen:

git clone https://github.com/KlipperScreen/KlipperScreen.git ~/KlipperScreen
~/KlipperScreen/scripts/KlipperScreen-install.sh

Each profile in deckhand-profiles declares one or more screens with their install steps under screens[].install; copy from there if your printer isn't covered above.

8. Evict vendor services

Same list S120 walks you through, just done in one go. The list comes from your profile's stock_os.services[]. On the Arco, the usual suspects:

# Block UDP discovery beacons
sudo systemctl disable --now makerbase-net-mods.service

# Stop vendor screen daemon (replaced by KlipperScreen)
sudo systemctl disable --now phrozen-screen-svc.service

# Vendor cloud bridge, definitely not needed
sudo systemctl disable --now makerbase-cloud.service

# Verify nothing's running you don't expect
systemctl list-units --type=service --state=running \
  | grep -Ev '^(klipper|moonraker|nginx|sshd|systemd|cron)'

9. Remove vendor files

S140 gives you a tickbox list. Doing it by hand, look up the profile's paths and rm -rf what you want gone. The GUI shows you a diff preview before deleting; on the shell side, please double-check each path before you commit.

# From phrozen-arco profile, stock_os.files[]:
sudo rm -rf /opt/phrozen/legacy-cgi/
sudo rm -rf /var/lib/phrozen/cloud-cache/
sudo rm    /etc/phrozen/branding.json
sudo rm -rf ~/printer_data/config.vendor-overrides/

10. Hardening (S150)

# Change the well-known SSH password
passwd

# Fix the 3-way time-sync conflict (chrony + ntpd + systemd-timesyncd)
sudo systemctl disable --now chrony ntp 2>/dev/null
sudo systemctl enable  --now systemd-timesyncd

# Turn off vendor UDP if it wasn't covered above
sudo ufw deny 8989/udp 2>/dev/null   # if ufw is installed

11. Fresh flash (Path B only)

Only if you're wiping the eMMC. The GUI does this with the elevated helper + the FlashService.write RPC; manually you'd Etcher or dd.

Triple-check your of= path

Wrong device and you wipe your laptop. Run lsblk or diskutil list with the card unplugged, then plug it in and run again. The letter that changed is your card.

# 1. Download Armbian for your printer's SoC (RK3328 for Arco)
wget https://dl.armbian.com/.../Armbian_24.x_Tinkerboard_jammy_current.img.xz
xz -d Armbian_24.x_Tinkerboard_jammy_current.img.xz

# 2. Confirm target device (CARD must be unmounted)
lsblk
sudo umount /dev/sdX*

# 3. Write, this takes ~5 min for an 8GB image
sudo dd if=Armbian_24.x_Tinkerboard_jammy_current.img \
        of=/dev/sdX bs=4M status=progress conv=fsync

# 4. Sync, eject, plug into printer, boot, then SSH and run all
#    of the above starting from §4.

12. Did it work?

Same as S910. Klipper should report Ready and Moonraker should answer the door:

# From your laptop:
curl -s http://printer.local:7125/printer/info | jq .result.state
# → "ready"

curl -s http://printer.local/ -o /dev/null -w '%{http_code}\n'
# → 200  (Mainsail/Fluidd serving)

If state is error, SSH in and tail Klipper's log: journalctl -u klipper -n 100. The wizard's debug-bundle creator (cli.html) collects the same files for you with redaction.

Rollback: restore the snapshot

If something went sideways and you took the snapshot in §3:

scp deckhand-snapshot-*.tar.gz mks@printer.local:/tmp/
ssh mks@printer.local '
  cd ~ && mv printer_data printer_data.failed-$(date +%s)
  tar -xzf /tmp/deckhand-snapshot-*.tar.gz
  sudo systemctl restart klipper moonraker
  rm /tmp/deckhand-snapshot-*.tar.gz
'
